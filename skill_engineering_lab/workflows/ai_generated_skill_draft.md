# Genshin Code Guide v2

> 基于 B站 + 第三方抓取实战工作流总结，v2 在 v1 基础上补充了失败处理和 SPA 兼容策略。

## 适用场景

使用本 Skill 处理以下任务：

- 抓取公开页面中的原神兑换码、直播兑换码、活动礼包码信息。
- 从页面标题、正文和发布时间中抽取兑换码、来源和时间信息。
- 判断兑换码是否仍可能有效，并说明判断依据。
- 汇总多个来源，去重并标注官方来源、半官方来源和第三方来源。
- 生成面向用户的安全提示，提醒不要点击可疑链接或提交账号密码。

## 不适用场景

以下任务不使用本 Skill：

- 获取、购买、交换或售卖私人兑换码。
- 绕过登录、验证码、反爬机制或访问非公开接口。
- 自动登录游戏账号、提交兑换、批量试码或刷码。
- 生成看似真实但未经来源核验的"最新可用兑换码"。

## 输入要求

Agent 开始前应确认：

- 目标来源列表：每个来源需有名称、URL、类型（official_community / official_social / official_site / third_party）。
- 如果用户只描述任务而未提供 URL，应先用搜索工具定位公开页面链接。
- 页面类型预判：静态 HTML、SPA（JS 渲染）、或存在公开 API。静态页面可直接抓取，SPA 页面应优先查找公开 API。

## 工作流程

1. **来源定位**：使用搜索工具定位官方和可信公开页面 URL，记录每个来源的名称、URL、来源类型。
2. **页面类别判断**：确认页面是静态 HTML、SPA（依赖 JS 渲染）还是存在公开 API。静态页面用 urllib 抓取；SPA 页面优先查公开 API。
3. **抓取与回退**：
   - 优先用公开 API 获取结构化数据。
   - API 返回风控（如 412）时，换用其他公开端点（如文章 API 替代动态 API）。
   - 所有可用途径均失败时，标记为"当前来源不适合自动抓取"，**不要尝试绕过风控**。
4. **文本抽取**：从 API 响应或 HTML 中提取标题、正文、发布时间和链接。
5. **兑换码抽取**：
   - 用正则匹配疑似兑换码（纯英文大写 + 数字，8-20 位）。
   - **注意**：Python 3 中 `\b` 在 CJK 字符边界可能失效，需额外处理中英文混排文本。
   - 过滤明显不是兑换码的订单号、UID、版本号、URL 片段。
6. **时间解析**：
   - 从正文附近抽取时间表达（"即日起""有效期至""直播结束后"等）。
   - 码中含日期线索（如 "20260812"）可用于推导 `valid_until`。
   - 将时间统一为带时区字段；无法确定时标记 `unknown`，不编造。
7. **去重与可信度**：
   - 同一兑换码按代码值去重，保留最可信来源。
   - **第三方来源永远标 `confidence=low`、`status=unverified`**。
8. **状态标注**：每条记录标注 `active_candidate` | `expired` | `unknown` | `unverified`。
9. **输出前自检**：逐条对照字段约定表，检查是否有冗余字段、错误标注或来源混淆。

## 输出要求

- 输出 Markdown 表格和 JSON 两种结构。
- 每条记录至少包含 11 个字段：`code`、`source_name`、`source_url`、`source_type`、`published_at`、`valid_from`、`valid_until`、`status`、`confidence`、`captured_at`、`evidence_text`。
- 不把第三方未核验信息写成官方结论。
- 不输出要求登录、付费、加群、下载 App 外安装包或填写账号密码的可疑链接。
- 如果没有找到可核验兑换码，明确说明"未在允许来源中找到可核验信息"。

## 自查清单

完成后检查：

- 是否只抓取公开页面。
- 是否记录抓取时间和来源 URL。
- 是否抽取并归一化生效时间、失效时间和发布时间。
- 是否把未知时间明确标记为 `unknown`。
- 是否区分官方、半官方和第三方来源。
- 是否对过期或无法验证的兑换码给出谨慎提示。
- 是否避免编造"最新可用码"。
- 是否逐条核对 source_name/source_type/source_url 三个字段一致性。
- 输出 JSON 字段是否严格符合约定（不多不少）。

## 失败处理

| 情况 | 处理方式 |
|:--|:--|
| 页面为 SPA，无公开 API | 尝试 web_fetch；若返回空白，标记为"不适合自动抓取" |
| API 返回 412/403 等风控 | 换用同站点其他公开端点；全部失败后记录原因，不绕过 |
| 正则无法匹配中文环境兑换码 | 改用文本搜索 + 手工提取，记录为已知限制 |
| 页面无时间信息 | `published_at`/`valid_from`/`valid_until` 均标 `unknown` |
| 多个来源对同一码有冲突信息 | 优先采信官方来源，在 `evidence_text` 中记录冲突 |
| 工具不可用（如浏览器自动化） | 回退到 API 优先策略，以课程环境为准 |
