# 智能资料整理助手 — 功能手册

> AI-powered Document Organizer | 基于大模型的资料自动整理与知识问答系统

---

## 一、项目概述

**智能资料整理助手**是一个 AI 驱动的资料管理工具，用户提交任意文档内容后，AI 会自动完成**分类、打标签、生成摘要、提取关键要点**，并支持基于已整理资料库的 **RAG 智能问答**。

### 核心价值

| 价值点 | 说明 |
|:--|:--|
| 自动整理 | 录入原始文本 → AI 自动分类 + 标签 + 摘要 + 要点 |
| 知识检索 | 关键词搜索已整理资料，快速定位 |
| 智能问答 | 基于资料库的 RAG 检索增强生成，回答附带来源引用 |
| 安全可控 | Token 鉴权 + 频率限制 + 内容过滤 + Token 预算控制 |

## 三、API 接口一览

所有接口需在 `Authorization` 头中携带 `Bearer <token>`。

| 方法 | 路径 | 功能 | 安全层 |
|:--|:--|:--|:--|
| `GET` | `/health` | 健康检查（无需 Token） | - |
| `GET` | `/` | API 导航信息 | - |
| `POST` | `/api/ingest` | 提交资料进行 AI 整理 | 鉴权 + 频率限 + 内容过滤 + Token 预算 |
| `GET` | `/api/documents` | 列出已整理资料（支持分页和分类过滤） | 鉴权 |
| `GET` | `/api/documents/{id}` | 查看单篇资料详情 | 鉴权 |
| `DELETE` | `/api/documents/{id}` | 删除资料 | 鉴权 |
| `POST` | `/api/search` | 关键词搜索资料 | 鉴权 |
| `POST` | `/api/ask` | 基于资料库的 RAG 智能问答 | 鉴权 + 频率限 + 内容过滤 |

### 请求/响应示例

**录入资料：**
```bash
curl -X POST http://127.0.0.1:8000/api/ingest \
  -H "Authorization: Bearer dev-local-token" \
  -H "Content-Type: application/json" \
  -d '{"title":"Python 异步编程","content":"asyncio 是 Python 3.4 引入的..."}'
```

**响应：**
```json
{
  "ok": true,
  "document_id": "a1b2c3d4e5f6",
  "title": "Python 异步编程",
  "category": "技术/编程",
  "tags": ["Python", "异步编程", "asyncio"],
  "summary": "本文介绍了 Python asyncio 异步编程的核心概念...",
  "key_points": ["事件循环机制", "async/await语法", "Task并发控制"],
  "token_usage": {"total_tokens": 342, "prompt_tokens": 180, "completion_tokens": 162},
  "request_id": "..."
}
```

**智能问答：**
```bash
curl -X POST http://127.0.0.1:8000/api/ask \
  -H "Authorization: Bearer dev-local-token" \
  -H "Content-Type: application/json" \
  -d '{"question":"异步编程的核心机制是什么？"}'
```

**响应：**
```json
{
  "ok": true,

  ],
  "token_usage": {"total_tokens": 512},

```

## 四、安全机制

| 层次 | 机制 | 说明 |
|:--|:--|:--|
| 认证 | Bearer Token | 所有数据接口需鉴权 |
| 频率限制 | 滑动窗口 20次/分钟 | 防止滥用 |
| Token 预算 | 单次请求 ≤ 4000 tokens | 控制成本 |
| 内容过滤 | 敏感词检测 | 阻止不合规内容 |
| 拒答机制 | RAG 不足时明确拒绝 | 避免幻觉 |

## 六、使用步骤

### 1. 配置环境
编辑 `.env` 文件：
```env
OPENAI_API_KEY=sk-你的真实Key
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o-mini
API_TOKEN=dev-local-token
```

### 2. 启动服务
```bash
# Windows
start.bat

# 或手动启动
python -m uvicorn src.app:app --host 127.0.0.1 --port 8000
```

### 3. 打开前端
双击 `index.html`，即可在浏览器中使用。

## 七、数据存储

```
data/documents/
├── index.json          # 元数据索引（分类、标签、时间等）
├── a1b2c3d4e5f6.json   # 文档1完整内容
```

- 零外部数据库依赖，纯文件存储
- 重启服务不丢数据
- 可通过「清空数据」按钮一键清理
