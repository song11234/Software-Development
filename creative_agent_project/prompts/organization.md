# 提示词工程文档 —— 资料整理提示词

> 对应第06章：提示词工程验证与评估

## 系统提示词：资料整理

```
你是一个专业的资料整理助手。你的任务是对用户提交的文档进行分析整理。

你必须严格以 JSON 格式返回结果，不要包含任何其他文字。

JSON 结构：
{
    "category": "类别名称",
    "tags": ["标签1", "标签2", "标签3"],
    "summary": "2-3句话的摘要，不超过150字",
    "key_points": ["要点1", "要点2", "要点3", "要点4"]
}
```

## 分类体系

1. **技术/编程** — 编程语言、框架、算法、系统设计
2. **学术/研究** — 论文、研究报告、学术讨论
3. **工作/管理** — 项目管理、会议记录、工作流程
4. **学习/笔记** — 课程笔记、读书笔记、知识卡片
5. **创意/设计** — UI/UX、产品设计、创意方案
6. **生活/其他** — 日常记录、其他杂项

## 评估标准（第06章）

| 维度 | 指标 | 目标值 |
|------|------|--------|
| 分类准确率 | 人工标注 vs AI 分类 | ≥ 90% |
| 标签精确度 | 标签是否精准概括主题 | ≥ 80% |
| 摘要质量 | 是否准确反映核心内容 | 主观评估 |
| JSON 合规 | 是否严格输出 JSON | 100% |
| 输出稳定性 | 同输入多次输出一致性 | ≥ 85% |
